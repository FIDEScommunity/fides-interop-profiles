# FIDES Interoperability Profile Matrix

An interactive matrix view for comparing interoperability profiles like DIIP, HAIP, EWC, and Swiyu.

## Overview

This repository contains:
- **JSON Schema v2** for interop profile definitions
- **Profile Data** in machine-readable format (v2 schema)
- **Crawler Script** for aggregating profile data
- **WordPress Plugin** for matrix display on websites

## Project Structure

```
interop-profiles/
├── profiles/                          # Profile JSON files
│   ├── interop-profile.diip-v4.json
│   ├── interop-profile.diip-v5.json
│   └── interop-profile.haip-v1.json
├── data/
│   └── aggregated.json                # Generated by crawler
├── schemas/
│   └── interop-profile.schema.json    # JSON Schema definition
├── src/
│   └── crawler/
│       └── index.ts                   # Data aggregation script
└── wordpress-plugin/
   └── fides-interop-matrix/          # WordPress plugin
```

## Quick Start

### 1. Add Profile Data

Place new interop profile JSON files in the `profiles/` folder:

```bash
profiles/interop-profile.<profile-id>.json
```

**Naming convention**: `interop-profile.<abbreviation>-<version>.json`

Examples:
- `interop-profile.diip-v4.json`
- `interop-profile.haip-v1.json`
- `interop-profile.ewc-v3.json`

### 2. Validate and Aggregate Data

```bash
npm install
npm run validate  # Validate profiles against schema
npm run build     # Generate aggregated.json + minified assets
```

This generates `data/aggregated.json` with all profiles.

### 3. Install WordPress Plugin

See [DEPLOYMENT.md](DEPLOYMENT.md) for complete deployment instructions.

**Quick install**:
1. Copy `wordpress-plugin/fides-interop-matrix/` to WordPress `wp-content/plugins/`
2. Ensure `aggregated.json` is in `assets/` folder
3. Activate plugin in WordPress Admin
4. Use shortcode: `[fides_interop_matrix]`

**Shortcode Options**:
```
[fides_interop_matrix profiles="diip-v5,haip-v1" theme="fides"]
```

## Matrix Features

- **Desktop**: Side-by-side comparison matrix with aligned columns
- **Mobile**: Tab-based view with swipe gestures
- **Visual Indicators**:
  - ✓ (green) = Supported
  - ✗ (red) = Not supported
  - Version info displayed next to checkmark
- **Toggle**: Hide rows where all profiles are unsupported
- **Responsive**: Works on desktop, tablet and mobile
- **FIDES Theme**: Consistent with FIDES community branding

## Profile ID Format

Profile IDs follow the format: `<abbreviation>-<version>`

Examples:
- `diip-v4`, `diip-v5`
- `haip-v1`
- `ewc-v3`
- `swiyu-v1`
